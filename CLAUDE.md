# whitneyesque.github.io — context for Claude Code sessions

This is the Astro portfolio site. New Claude Code sessions opening here should skim this first.

---

## What this is

Whitney's portfolio site, deployed at **https://whitneyesque.github.io**. Built with Astro 5 + content collections. Every PR gets a Cloudflare Pages preview URL. Main also deploys to Cloudflare Workers (added April 2026 by Cloudflare's auto-config) and GitHub Pages — both deploy paths run, both work.

The case-study **content** is generated by an agent pipeline in a separate repo: **`whitneyesque/Agent-Land`**. See its `CLAUDE.md` for that side. This repo holds the layout, components, and final MDX files.

---

## Cases live on main (April 2026)

`src/content/cases/` contains:
- `agetech.mdx` — featured on /work
- `ai-lx.mdx` — anonymized CarMax case
- `allied.mdx`
- `ferguson-cx.mdx`

## V4 design system

Components in `src/components/` (in addition to existing PullQuote / StatRow / InsightCallout / SiteHeader / SiteFooter):
- `BotanicalDivider.astro`
- `DarkBand.astro`
- `ProcessTimeline.astro`
- `SpecimenSignoff.astro`
- `Placeholder.astro` — scaffolding only, used by visual-director agent for missing-asset markers; should be removed before publishing

Layout: `src/layouts/CaseLayout.astro`. Renders frontmatter title (with `set:html` so inline `<em>` works), client meta, a "Back to Work" link, and a slot for the MDX body.

Schema: `src/content/config.ts`. Required: title, client, year, role, timeline, tags, summary, status. Optional: team, specimen, specimenSignoff, hero_image. Status enum: `'draft' | 'published'`.

CSS: case-specific styles appended to `public/style.css` in a "Case study V4 additions" block.

## Work page

`src/pages/work.astro`. Currently hand-curated (not auto-generated from collection). Has:
- One featured card (agetech) with conditional Read Case Study link
- A grid of 5 cards: Ferguson Data, Allied, SFPL, ai-lx, ferguson-cx — each with a thumbnail quote, summary, tags, and a conditional Read Case Study link gated on `published.has(slug)`
- One "More on the way" placeholder

The thumbnail quotes are real research quotes pulled from Miro boards. See `agent-land/CLAUDE.md` for how they were sourced.

When adding a new case to the work page, follow the existing card pattern. The Read Case Study link uses `published.has('<slug>') && <a href="/cases/<slug>.html" class="btn">Read Case Study</a>` so it auto-renders only when the case file exists.

## Deploy

Two paths run on every push to main; both succeed:
- **GitHub Pages** via `.github/workflows/` deploy workflow → `https://whitneyesque.github.io`
- **Cloudflare Workers** via `wrangler.jsonc`

PR previews via the **PR Preview (Cloudflare Pages)** workflow → posts a preview URL comment on each PR.

## Conventions

- **Slug naming**: lowercase ASCII + hyphens. The Agent-Land composer enforces this; respect it here too.
- **Anonymized cases**: client name in frontmatter is generic (e.g., "A national automotive retailer" for ai-lx). Body prose has bracketed substitutions (`[the client]`) inside direct quotes; descriptive prose uses paraphrase ("the retailer's design team").
- **`.claude/`** is in `.gitignore` — Claude Code launch configs and worktrees stay local.
- **Botanical specimens**: each case may have a `specimen` (cover image) and `specimenSignoff` (end-of-case ornament). Optional — both can be omitted.

## Common asks in a fresh session

- "Update a case's prose" → edit `src/content/cases/<slug>.mdx`, push, preview, merge
- "Add a new case to the work page" → edit `src/pages/work.astro`, copy an existing `<article class="project-card">` as template
- "Why isn't /cases/X working?" → check that `src/content/cases/X.mdx` exists on main with `status: 'published'`
- "Update a thumbnail quote" → edit the `<p class="thumb-quote">` and `<span class="thumb-cite">` in `src/pages/work.astro`
